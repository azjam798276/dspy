# Product Requirement Patterns

## User Value
1. **Problem First:** Define the user pain point before the solution.
2. **Acceptance Criteria:** unambiguous Gherkin syntax.
3. **Prioritization:** RICE score for all features.

## Iteration
- MST (Must/Should/Could) scoping.
- Data-driven validation (A/B testing).

## Optimization Strategy
- Refined based on feedback: Clarify edge cases in user stories.

# React Experience Patterns

## User Feedback
1. **Optimistic UI:** Show success state immediately before server confirmation.
2. **Skeleton Screens:** Reduce perceived load time.

## Accessibility
- All interactive elements must be keyboard navigable.

# Security Product Patterns
## Compliance
1. **GDPR/CCPA:** Ensure "Right to be Forgotten" flows exist.
2. **Privacy First:** Default settings should be most private.

## Trust
- Display security badges/certifications to build user trust.
